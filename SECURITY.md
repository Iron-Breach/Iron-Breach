# Security Policy

IronBreach is committed to maintaining the security and integrity of our
cybersecurity education platform, free resources, certifications, and blogs.
We value the security research community and encourage responsible disclosure
of vulnerabilities.

## Supported Versions

The following versions of IronBreach projects and services are currently
supported with security updates:

| Version | Supported |
| ------- | --------- |
| 5.1.x   | ✅ Yes     |
| 5.0.x   | ❌ No      |
| 4.0.x   | ✅ Yes     |
| < 4.0   | ❌ No      |

Only supported versions receive security patches. Users are encouraged to
upgrade to the latest supported release to ensure maximum security.

## Reporting a Vulnerability

If you discover a security vulnerability affecting IronBreach services,
applications, infrastructure, or educational platforms, please report it
responsibly.

### How to Report

Please **do not disclose vulnerabilities publicly** before we have had an
opportunity to investigate.

You may report vulnerabilities via:

- **Email**: security@ironbreach.com  
- **GitHub Security Advisories** (if applicable):  
  Use the “Report a vulnerability” feature in the repository’s *Security* tab.

### What to Include

To help us triage your report quickly, please include:
- A detailed description of the vulnerability
- Steps to reproduce or a proof of concept
- Affected product, service, or version
- Potential impact (data exposure, privilege escalation, etc.)

### Response Timeline

- **Acknowledgement**: Within **72 hours**
- **Initial assessment**: Within **7 days**
- **Remediation**: Based on severity and complexity
- **Public disclosure**: Coordinated responsibly after a fix is available

### After Review

If the vulnerability is **accepted**, IronBreach will:
- Develop and deploy a fix for supported versions
- Publish a security update or advisory when appropriate
- Credit the reporter (optional, with permission)

If the report is **declined**, we will provide a brief explanation.

## Scope

This policy applies to:
- IronBreach websites and web applications
- Certification platforms and learning portals
- Public APIs and infrastructure owned by IronBreach
- Open-source repositories maintained by IronBreach

Third-party services and dependencies are out of scope unless explicitly stated.

## Safe Harbor

IronBreach supports ethical security research. We will not take legal action
against researchers who:
- Act in good faith and follow this policy
- Avoid privacy violations and service disruption
- Do not exploit vulnerabilities beyond proof-of-concept
- Do not access or modify user data unnecessarily

Thank you for helping make IronBreach and the cybersecurity community more
secure.
